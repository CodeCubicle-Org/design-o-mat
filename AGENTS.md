# Agents

This repository is **design-o-mat** ([GitHub](https://github.com/CodeCubicle-Org/design-o-mat)).

- **Coding / build agents:** follow this file and project conventions for how to implement code. **Repository scripts and automation** use **Node.js** (ES modules, **v24 or newer** per `package.json` `engines`); do **not** add Python-based tooling to this repo.
- **Design / UI agents:** follow **`DESIGN.md`** in this repo (or the project root where the product spec lives) for look-and-feel, tokens, and component rules.

Authoring or refreshing `DESIGN.md` starts with **Step 1**: fill the shared intake [`design-definition-form.md`](docs/design-definition-form.md) (designers + developers—colors, typography including Title 1, layout, components, sign-off), then use the phased workflow in [`template-DESIGN.md`](docs/template-DESIGN.md). Optional Cursor **project skills** live under [`.cursor/skills/`](.cursor/skills/) (`design-md-*`) so individual steps (scope lock, token capture, validate, previews, ship, format reference) can be applied on demand.
